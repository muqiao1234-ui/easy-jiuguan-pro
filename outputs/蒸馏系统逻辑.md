# Easy酒馆Pro · 蒸馏系统全链路逻辑（2026-07-14 最终版）

---

## 一、系统概述

蒸馏系统（Distillation）将对话历史压缩为**记忆结晶**（distilled node），在上下文组装时以 system 消息注入，使 AI 在有限的 token 预算内记住更长的对话历史。

核心流程：**触发 → 扫描世界书 → 查询前序记忆 → 调用蒸馏AI → 生成记忆结晶 → 归档旧消息**。

---

## 二、配置项（DistillationConfig）

| 字段 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `triggerThreshold` | `number` | 10 | 触发阈值：未归档消息数 ≥ 此值时触发蒸馏 |
| `retainRecentCount` | `number` | 3 | 滑动窗口：强制保留最近 N 条消息不参与蒸馏，保持即时对话上下文 |
| `autoTrigger` | `boolean` | false | 是否在每次 AI 回复后自动检查并触发蒸馏 |
| `distillationPrompt` | `string` | 内置模板 | 自定义蒸馏提示词，`{dialogue}` 替换为对话文本 |

---

## 三、触发机制

### 3.1 自动触发（每条 AI 回复之后）

```
未归档消息数 ≥ triggerThreshold
    │
    ├─ 过滤条件：排除 distilled / system / scribe 角色的节点
    │
    ├─ 按 timestamp 升序排序
    │
    ├─ 滑动窗口切片：
    │   toDistill = sorted[0 .. length - retainRecentCount]
    │   （最旧的 (总数-保留) 条参与蒸馏，最新 retainRecentCount 条留在现场）
    │
    └─ 若 toDistill.length > 0 → 执行蒸馏
```

**触发时机**：在 Galgame 引擎 / 状态书处理完成后触发，避免并发。

### 3.2 手动触发（用户点击「蒸馏」按钮）

```
用户点击 → 查询当前会话全部节点 → 过滤 + 排序 + 切片
    │
    ├─ 既往不咎逻辑：
    │   maxBatch = min(triggerThreshold, total - retainRecentCount)
    │   只取最旧的 maxBatch 条进行蒸馏
    │   → 避免「20条未归档，手动点一次全杀」
    │
    ├─ 切片后若 toDistill.length === 0 → 报错提示
    │
    └─ 执行蒸馏（与自动触发共用同一 performDistillation）
```

**既往不咎的关键设计**：手动蒸馏的上限是 `triggerThreshold`（一个批次），不是全部未归档消息。用户可以多次点击，每次消化一个批次。

## 四、蒸馏前准备（performDistillation 内部）

### 4.1 查找上一轮记忆结晶

```typescript
const prevDistilled = allNodes
  .filter(n => n.role === 'distilled')
  .sort((a, b) => b.timestamp - a.timestamp)[0];
```

取最新的 `distilled` 角色节点，将其 `content` 作为 `prevDistilledContent` 传入。

### 4.2 扫描蒸馏区间的世界书

```typescript
const wbEntries = await scanWorldBook(
  characterA.worldBookId,
  toDistill,              // 仅扫描被蒸馏这批消息
  maxWorldBookEntries     // 上限
);
```

扫描被蒸馏消息区间内激活的世界书条目，用于两个目的：
- **蒸馏AI输入**：附带完整正文，帮助AI理解世界观
- **记忆结晶存储**：仅保存名词列表，节约token（详见第五节）

## 五、蒸馏AI调用

### 5.1 Messages 四段结构

```
[1] user: 蒸馏提示词（首部指令）
         "请将以下对话内容浓缩为简洁的摘要..."

[2] user: 上一轮记忆结晶（如有）
         "这是上一轮蒸馏产出的记忆结晶，请在其基础上续写，
          保持记忆连贯性，不要重复已有内容：\n\n" + prevDistilledContent

[3] user: 本区间对话 + 激活世界书
         "以下是本轮需要蒸馏的对话内容（含区间激活的世界书信息）：
          
         角色A: xxxxx  
         角色B: xxxxx
         
         --- 本区间激活的世界书条目 ---
         [词条名] 完整正文..." 

[4] user: 蒸馏提示词（尾部强化）
         重复步骤[1]的提示词，强化输出格式约束
```

### 5.2 API 参数

- `model`: 蒸馏模型 ID
- `messages`: 上述四段结构
- `stream: false`（非流式，一次性返回）

## 六、记忆结晶生成与存储

### 6.1 内容拼装

```
AI 返回的摘要正文
  +
(附带：区间激活的世界书条目表)   ← formatWorldBookAppendix()
[词条1] [词条2] [词条3]         ← 仅名词，不包含正文，节约token
```

### 6.2 生成格式

使用 `tplDistilledNodePrefix` 模板（默认 `📝 记忆 第1轮-第{total}轮：{summary}`），`{total}` 为被蒸馏消息总数，`{summary}` 为上述拼装内容。

### 6.3 节点属性

| 属性 | 值 |
|------|-----|
| `role` | `'distilled'` |
| `senderName` | `'记忆结晶'` |
| `timestamp` | 被蒸馏的最后一条消息时间戳 + 1ms |
| `isArchived` | `false`（保持未归档，参与上下文组装） |
| `conversationId` | 继承自被蒸馏消息 |

### 6.4 归档操作

仅对被蒸馏的这批旧消息执行 `batchUpdateNodes(..., isArchived: true)`。滑动窗口保留的近期消息不受影响。

## 七、上下文组装中的蒸馏节点注入

在 `assembleContext` 中（`src/utils/context.ts`），蒸馏节点作为**高优先级内容**注入：

```
[system] 角色 systemPrompt
[system] 世界书条目 1
[system] 世界书条目 2
[system] 记忆结晶 1  ← 含尾部粘连的世界书名词，可能被重新扫描激活
[system] 记忆结晶 2
[system] 记忆结晶 3
...
[user]   最近的对话消息
[assistant] ...
[system]  身份锚点
```

### 多轮蒸馏拼接效果

多个记忆结晶按时间正序自然拼接，每个独立携带自己的世界书附录：

```
记忆结晶a（第1-7轮） + (附带：[少女前线] [格里芬])
记忆结晶b（第8-14轮） + (附带：[铁血工造] [AR小队])
记忆结晶c（第15-21轮） + (附带：[帕拉蒂斯] [安洁莉娅])
```

→ 下次对话时世界书扫描器会命中 `[少女前线]` `[格里芬]` 等关键词，自动重新注入对应条目，避免大段世界书上下文丢失。

## 八、记忆回廊（Memory Corridor）

对话工具栏中蒸馏按钮旁的 `📖 记忆回廊` 按钮：

- 点击弹出 Modal，按时间正序展示所有记忆结晶
- 每条显示：编号 `#N`、时间戳、完整内容（含世界书附录）
- 无记忆结晶时按钮禁用，弹窗显示空态提示
- 让用户在浓缩对话的历史摘要上快速回顾剧情
