# 状态书 / Galgame 引擎 — 提示词拼装全链路

> 审查日期：2026-07-14 · 项目：`git版本\easyjiuguanpro`

---

## 概述

项目有两套状态书引擎：

| 引擎 | 类型 | 存储 | 注入 | 触发间隔 |
|------|------|------|------|----------|
| **文本状态书** (`scribe`) | 自由文本 | `node.scribeUpdate.{rawText, isEnabled, mode}` | system 消息注入 | 默认 5 轮，用户可调 (`scribeTriggerInterval`) |
| **Galgame 数值引擎** (`galgame`) | 结构化 JSON | `node.galgameData.{name, health, mood, vigilance, surfaceAffinity, hiddenAffinity}` | system 消息注入（非对称模糊化） | 固定 2 轮 (`GALGAME_TRIGGER_INTERVAL = 2`) |

---

## 1. 触发链路

```
用户发消息 / 旁听触发
  → AI 回复完成 (addMessageNode)
  → (async fire-and-forget, 串行)
     ├─ 1. 状态书触发判断 (useChat.ts:524-569)
     │   ├─ scribeEnabled + scribeModelId 未配置 → 跳过
     │   ├─ 计算 assistantCount（该角色已回复轮数）
     │   ├─ interval = galgame 模式 ? 2 : scribeTriggerInterval(默认5)
     │   ├─ shouldTrigger = assistantCount > 0 && assistantCount % interval === 0
     │   │
     │   └─ shouldTrigger && 模式匹配：
     │       ├─ scribeMode=charA + aiRole=charA → 执行（针对角色A）
     │       ├─ scribeMode=charB + aiRole=charB → 执行（针对角色B）
     │       ├─ scribeMode=auto → 执行（任意角色）
     │       │
     │       └─ 引擎分支：
     │           ├─ scribeEngine=galgame → triggerGalgameEngine(...)
     │           └─ scribeEngine=text    → triggerScribeSummary(...)
     │
     └─ 2. 自动蒸馏 (Galgame/scribe 完成后)
        └─ autoTriggerDistillation + 未归档 ≥ triggerThreshold(10) → performDistillation
```

**输入范围**：`nonSystemNodes.slice(-deps.recentRounds)` （默认最近 20 轮）

## 2. 文本状态书 (`triggerScribeSummary`)

### 入口：`useChat.ts:100`

### Prompt 拼装

```
messages = [
  { role: "system", content: scribeSystemPrompt || SCRIBE_SYSTEM_PROMPT },
  { role: "user",   content: 纯对话文本 },
]
```

**System Prompt（默认）**：
> 你是不参与对话的独立客观书记官。请根据以下最近的对话内容，冷酷、客观地更新并精简当前的环境状态、时间、地点、交互物品状态及角色间的好感度变化。不要输出任何对话，仅输出最新的状态面板。

**User 消息**：`recentRounds` 条最近消息中，仅取 `user/charA/charB` 角色，格式：
```
核桃: xxx
花生: xxx
user: xxx
```

### API 参数
- `stream: false`
- temperature/topP: 从模型配置读取（`...buildSamplingParams`）
- **无角色提示词注入**
- **无历史状态书注入**

### 输出处理
- 取 `choices[0].message.content`
- 写入 `node.scribeUpdate = { rawText, isEnabled: true, mode }`

### ⚠️ 特点
- **上下文绝对隔离**：仅接收 system prompt + 纯对话文本，不知道角色设定
- **无历史记忆**：每次从零开始总结，不依赖上一次状态书结果
- **无字符限额**：不设 `max_tokens`

## 3. Galgame 数值引擎 (`triggerGalgameEngine`)

### 入口：`useChat.ts:186`

### Prompt 拼装

```
messages = [
  { role: "system",    content: galgamePrompt (用户自定义或默认) },
  { role: "user",      content: 角色卡 injection (如果有 systemPrompt) },  ← 可选
  { role: "assistant", content: 历史状态书1 (序列化) },                    ← 可选
  { role: "assistant", content: 历史状态书2 (序列化) },                    ← 可选
  { role: "user",      content: 最近2轮对话 (清洗后的纯文本) },
]
```

### 逐层解释

**① System Prompt（galgameEngine.ts:8-30）**：
```
你是不参与对话、绝对冰冷客观的 Galgame 游戏后台数值引擎。
【重要】禁止输出任何推理过程、分析步骤、思考内容。直接输出最终 JSON 结果。

当前目标角色卡名字为：${targetCharacterName}

上下文说明：
- 如果有"角色提示词"→ 严格以该角色提示词为准来推导情感数值
- 如果有"历史状态书"→ 基于历史趋势延续性地调整
- 最后一条 user 消息是当前 2 轮对话

六维度定义：
1. name: 严格输出 "${targetCharacterName}"
2. health: 生理状态(良好/疲惫/轻伤/不适)
3. mood: 表面情绪(傲娇/欣喜/强颜欢笑/冷漠/羞怯)
4. vigilance: 警惕度 0-100
5. surface_affinity: 表好感 -100~100
6. hidden_affinity: 里好感 -100~100

必须输出严格单行 JSON:
{"name":"${targetCharacterName}","health":"健康度","mood":"表面心情","vigilance":0,"surface_affinity":0,"hidden_affinity":0}
```

**② 角色卡注入（可选，仅当角色有 systemPrompt）**：
```
{这是当前角色的角色提示词，状态书请以此角色逻辑判断感情}
<角色的完整 systemPrompt>
```
格式由 `tplGalgameCharInjection` 控制（可在高级提示词设置中编辑）。

**③ 历史状态书（可选，最近 2 条）**：
由 `collectRecentGalgameStates()` 从历史消息中抓取同名角色的 galgameData，序列化为：
```
name:曼城茶座 | health:良好 | mood:羞怯 | vigilance:45 | surfaceAffinity:30 | hiddenAffinity:55
```

**④ 当前对话（最近 2 轮 ≈ 4 条消息）**：
仅取 `user/charA/charB`，经 `cleanDialogueText()` 清洗（去 Emoji、去 Base64、去 HTML 标签）。

### API 参数
- `stream: false`
- **固定** `temperature: 0.3, top_p: 0.85`（不用模型的配置）
- **专设** `max_tokens: GALGAME_MAX_TOKENS = 2000`
- **不传** `reasoning_effort`

### 输出处理
- 优先取 `choices[0].message.content`
- content 为空 → 取 `reasoning_content`
- 仍为空 → 取 `choices[0].text`
- 调用 `parseGalgameResponse(raw, charName)` 解析 JSON
- 多策略容错：直接 parse → 正则提取 `{...}` → 去换行后 parse
- 名字校验：若模型返回的名字 ≠ 期望名字，强制修正
- 写入 `node.galgameData`

## 4. 上下文注入链路（context.ts）

### 时间线注入

在 `assembleContext()` 中，每条 assistant/对方角色消息后紧跟其 `scribeUpdate` / `galgameData`，以 **system 角色**注入：

**文本状态书**：
```
[system] [状态书更新]\n{scribeUpdate.rawText}
```
模板：`tplStateBookPrefix`（`DEFAULT_TPL_STATE_BOOK_PREFIX = '[状态书更新]\n{content}'`）

**Galgame — 情况 A（属于当前回戏角色）**：
```
[system] [系统环境提示] 角色 [曼城茶座] 当前身体状态为(良好)，外在情绪表现为(羞怯)。对方明显对真正的user表现出好感与亲近，但隐约可以察觉到对方潜意识里对真正的user有深层的依赖。
```
> 注意：好感度数字被模糊化为自然语言描述，不让主 AI 看到具体数值（防止作弊/谄媚）。

**Galgame — 情况 B（属于旁观/NPC角色）**：
```
[system] [当前场景NPC状态] 现场的另一位角色 [花生] 当前身体状态为(疲惫)，外在情绪表现为(冷漠)。对方与真正的user保持着礼貌的距离，对方内心深处似乎并不排斥真正的user的存在。
```

### Token 预算

注入前会估算 `scribeUpdate + galgameData` 的 token 开销，与消息本体一起计入 `usedTokens`，超出 `maxContextTokens` 时一起截断。

## 5. 两套引擎对比总结

| 维度 | 文本状态书 | Galgame 数值引擎 |
|------|-----------|-----------------|
| **角色认知** | ❌ 不知道角色设定 | ✅ 注入完整角色卡 systemPrompt |
| **历史记忆** | ❌ 每次从零开始 | ✅ 读取最近 2 条历史 galgameData |
| **触发间隔** | 用户可调 (默认 5) | 固定 2 轮 |
| **输入范围** | recentRounds 条 (默认20) | 最近 2 轮 (≈4条) |
| **输出格式** | 自由文本 | 严格六维 JSON |
| **temperature** | 模型配置 | 固定 0.3 |
| **max_tokens** | 无限制 | 2000 |
| **思考模式** | 跟随主模型 | 不传（可考虑加） |
| **注入到主对话** | system 消息原文 | system 消息模糊化描述 |
| **防作弊** | ❌ | ✅ 好感数字 → 模糊描述 |

## 6. 潜在优化点

| # | 问题 | 建议 |
|---|------|------|
| 1 | 文本状态书不知道角色设定，总结质量依赖 prompt 本身 | 参考 Galgame 模式注入角色卡 systemPrompt |
| 2 | 文本状态书无历史记忆，每触发一次就从头写 | 传入上次状态书内容作为参考 |
| 3 | Galgame 固定 0.3 temperature（偏低），对 GLM-4.7-Flash 可能太僵 | 考虑开放温度配置或跟随模型配置 |
| 4 | Galgame 不传 `reasoning_effort`，思维链能力未利用 | 添加 `reasoning_effort: 'medium'` |
| 5 | 文本状态书无 `max_tokens` 限制，回怼可能过长 | 加 `max_tokens: 1000` |
| 6 | Galgame 的 `gpt-5.1/claude` 模型可能不接受 temperature/top_p | 已在 `buildSamplingParams` 处理无设置模式 |
