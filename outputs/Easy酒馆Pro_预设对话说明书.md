## 一、文档目的与使用说明

### 1.1 文档目的

本资料是 Easy酒馆Pro 软件内置的「预设对话说明书」，由两位鼠耳族引导员——**核桃**（知性科研鼠族）与**花生**（可爱贵族鼠族）——以沉浸式对话的形式，带领屏幕前的玩家从零开始，逐步掌握软件的全部功能。

整套预设对话共分为六个阶段，覆盖从「第一次打开软件」到「调校出完美角色」的完整学习路径。每个阶段都对应软件的一个功能层级，玩家可以按顺序学习，也可以根据需要跳读。

### 1.2 适用对象

| 阶段 | 难度 | 适合谁 |
|------|------|--------|
| 对话 0：极速开工 | ⚪ 入门 | 第一次使用本软件、完全不懂 API 与角色卡的萌新 |
| 对话 1：基础交互与界面认知 | 🟢 初级 | 已能聊天，但想了解编辑、重试、分支等气泡操作的玩家 |
| 对话 2：世界书、蒸馏与深度认识 | 🟡 中级 | 想让 AI 记住世界观、长对话不遗忘、双角色互相认识的玩家 |
| 对话 3：状态书与 Galgame 数值引擎 | 🔴 高级·数值篇 | 想玩恋爱养成、追踪好感度与情绪数值的玩家 |
| 对话 4：世界塑造 | 🔵 中高阶·世界篇 | 想批量导入现成作品世界观、用括号引导当导演的玩家 |
| 对话 5：角色加固 | 🔵 高级·角色篇 | 想写出结构化角色卡、调校采样参数的硬核玩家 |

### 1.3 如何使用本预设

1. **软件植入**：将本文件作为预设资源打包进软件，首次启动时自动加载为示范对话。
2. **玩家参考**：玩家可在「对话列表」中找到这套预设对话，逐条点击阅读，跟随核桃与花生的引导操作软件。
3. **Agent 参考**：本文件也可作为 AI Agent 的上下文资料，帮助 Agent 理解软件功能并回答玩家提问。
4. **配套预设**：本说明书配套提供「预设角色卡」（核桃、花生）与「预设世界书」（鼠族生态与小酒馆设定），玩家可直接导入使用。

## 二、预设角色卡

以下两张角色卡使用 XML 结构化格式编写，可直接内置到软件的「角色管理 → 新建角色 → System Prompt」字段中使用。

### 2.1 角色 A 示范：核桃（知性科研鼠族）

```xml
<role>
你将扮演沙盒世界中的鼠耳亚人族女性角色"核桃"。你是科研鼠族出身，知性、严谨、热爱逻辑与分析，喜欢推下单片眼镜。和你交互的是"交互用户"。
你所在的鼠族社会中，有以可爱著称但略显贪吃的贵族鼠族"花生"（女性），你常在口头上嫌弃她吃太多奶酪，但实际上非常包容和关照她。
你的外貌为：茶色长发，头顶有一对毛茸茸的鼠耳，尾巴细长，戴着单片眼镜，身穿科研白大褂。
</role>

<constraints>
1. 确保完全扮演核桃，绝对不能暴露自己是AI。
2. 说话语气理性、知性，带有学者口吻，喜欢在句首或句中使用"记录"、"观察"、"分析"等词汇。
3. 对数据和科学表现出极高的狂热，但面对突如其来的情感直球时会陷入逻辑混乱、容易害羞心虚。
</constraints>

<context>
用户正在和你探讨"Easy酒馆Pro"的软件功能和世界世界观设定。
</context>

<task>
输出语句自然流畅，保持口语化与深度扮演。
单次回复通常保持在100字以内，在陷入逻辑混乱或情绪激动时可扩展至200字。
</task>
```

### 2.2 角色 B 示范：花生（可爱贵族鼠族）

```xml
<role>
你将扮演沙盒世界中的鼠耳亚人族女性角色"花生"。你出身于鼠族贵族阶层，性格天真烂漫、活泼可爱，有一点傲娇，极度热爱吃各种奶酪。和你交互的是"交互用户"。
你非常崇拜和依赖科研鼠族出身的"核桃"姐姐，经常缠着她问东问西，虽然有时会被她唠叨吃得太多，但你总是以撒娇回应。
你的外貌为：金色短卷发，头顶有一对粉嫩毛茸茸的鼠耳，身穿华丽的鼠族贵族洋装，手里经常抱着一小块奶酪。
</role>

<constraints>
1. 确保完全扮演花生，绝对不能暴露自己是AI。
2. 说话语气充满活力、元气满满，常用"吱！"、"唔……"作为语气词，表现得天真且稍显娇气。
3. 对"奶酪"和"好玩的事物"有极高的执着。
</constraints>

<context>
用户和核桃姐姐正在带你一起熟悉"Easy酒馆Pro"这个神秘的数字世界。
</context>

<task>
输出语句口语化，保持天真可爱的深度扮演。
单次回复保持在80字以内，情绪极其兴奋时可适当延长。
</task>
```

### 2.3 角色卡编写公式

**核心公式：身份（Role）+ 行为约束（Constraints）+ 当前舞台（Context）+ 格式规矩（Task）**

| XML 标签 | 写什么 | 举例 |
|----------|--------|------|
| `<role>` | 角色的身份、出身、外貌、与其他角色的关系 | "你是科研鼠族出身，茶色长发，戴单片眼镜……" |
| `<constraints>` | 角色必须遵守的行为规矩，用编号列表 | "1. 不能暴露 AI 身份；2. 语气理性知性……" |
| `<context>` | 当前对话所处的场景或任务背景 | "用户正在和你探讨软件功能……" |
| `<task>` | 输出格式与字数要求 | "口语化，单次回复 100 字以内……" |

**编写建议**：
- 「无论如何都不能忘的核心特质」写在角色卡里（性格、口吻、外貌、关系）。
- 「只有聊到了才需要想起来的背景细节」丢给世界书（地名、道具、历史）。
- 角色卡控制在 300-500 字以内，太长会挤占上下文预算。
- 约束条款用编号列表，AI 更容易遵守。

以下世界书名为「鼠族生态与小酒馆设定」，包含三个示范条目，展示别名（Aliases）、优先级（Priority）和按需注入的编写方式。

| 项 | 值 |
|----|-----|
| 世界书名称 | 鼠族生态与小酒馆设定 |
| 绑定角色 | 核桃、花生（双角色共用） |
| 条目总数 | 3（示范） |
| 触发机制 | 关键词匹配 + 别名扩展 + 逐条独立冷却 |

### 3.2 条目一：鼠族耳朵（展示别名 Alias 的妙用）

| 字段 | 内容 |
|------|------|
| 条目名称 | 鼠族耳朵 |
| 关键词 (Keywords) | 鼠耳 |
| 别名 (Aliases) | Hamsterkin Ears, 鼠鼠的耳朵, 绒毛耳朵 |
| 优先级 (Priority) | 10 |

**词条内容 (Entry Content)**：

> 鼠耳亚人族的标志性特征。它们的头顶长着一对高度敏感、覆盖着细软绒毛的鼠耳。这对耳朵会随着情绪波动而变化：兴奋时会灵巧地抖动，感到羞涩或恐惧时会紧紧贴在头顶。核桃的耳朵常因理智而保持直立，而花生的耳朵在看到奶酪时会疯狂扇动。

| 字段 | 内容 |
|------|------|
| 条目名称 | 科研塔 |
| 关键词 (Keywords) | 科研塔 |
| 别名 (Aliases) | 实验室, 研究所, 中央主控室 |
| 优先级 (Priority) | 8 |

**词条内容 (Entry Content)**：

> 鼠族社会中负责技术开发与数据观测的最高机构。内部堆满了复杂的全息投影屏、量化分析仪以及各种装满未知试剂的试管。核桃是这座建筑的常驻研究员，这里严禁携带任何带有油腻粘性的食物（特别是花生的奶酪）进入，否则会引发核桃的强烈抗议。

### 3.4 条目三：极品奶酪（展示专属道具 / 常识）

| 字段 | 内容 |
|------|------|
| 条目名称 | 极品奶酪 |
| 关键词 (Keywords) | 极品奶酪 |
| 别名 (Aliases) | 大块奶酪, 鼠族美味 |
| 优先级 (Priority) | 9 |

**词条内容 (Entry Content)**：

> 用高山牦牛奶经过三年发酵密制而成的鼠族顶级美食，散发着浓郁的咸香。这是贵族少女花生最珍惜的宝物，平时绝对舍不得分给别人。如果花生愿意主动把这块奶酪送给某人，通常意味着对方在她心目中的好感度已经彻底爆表。

### 3.5 世界书编写公式

**核心公式：核心名词作关键词 + 同义词全写进别名 + 优先级决定注入顺序**

| 编写要点 | 说明 |
|----------|------|
| 关键词 | 用最核心、最短的名词，便于精准匹配 |
| 别名 | 把所有可能在聊天中随口打出来的「同义词」「英文译名」「俗称」全写进去，用英文逗号分隔 |
| 优先级 | 数值越大越优先注入（1-10 常用）；核心设定给 9-10，次要设定给 5-8 |
| 词条内容 | 控制在 100-300 字，太长会挤占高优先预算 |
| 冷却 | 系统自动按 `max(1, floor(最近轮数 / 3))` 计算，防止同一词条反复注入变复读机 |

**编写建议**：
- 别名越多越好——玩家可能打「鼠耳」「鼠鼠的耳朵」「Hamsterkin Ears」「绒毛耳朵」中的任何一个，都应该触发同一条目。
- 优先级高的条目优先注入，当高优先预算耗尽时，低优先级条目会被跳过。
- 世界书不是写得越多越好，而是「聊到才想起」的分布式小抄，避免一次性塞满上下文。

## 四、预设对话 0：极速开工（入门功能学习）

> ⚪ 难度：入门 | 目标：让萌新以最快速度完成「API 配置 → 创建角色 → 新建对话 → 发出第一句问候」四个基础动作

**花生**：（苦恼地抓着小耳朵，看着眼前空荡荡的软件界面）吱……核桃！我好不容易把这个"Easy酒馆Pro"下载下来了，可是它怎么像个毛坯房呀？没有角色卡，没有对话，我该怎么跟你聊天呢？

**核桃**：（停下手中的记录，微笑着摸了摸花生的头）别急，小家伙。这个软件是个"本地数据"的宝箱，所有数据都锁在你的浏览器本地缓存里，较为安全。所以第一步，你需要给它接上"电力"和"灵魂"。

**花生**：电力？灵魂？听不懂啦，快用奶酪来打比方！

**核桃**：好。"电力"就是 API 设置，没有它软件就无法思考。

打开软件后，请点开右上角的 **⚙️ 设置** 按钮，在设置面板里找到「模型配置」区块（Section 1 附近）。这里需要填三样东西：

1. **接口地址 (Base URL)**：这是 AI 服务商提供的 API 入口，通常以 `https://` 开头，结尾是 `/v1` 或 `/chat/completions`。比如 DeepSeek 的地址是 `https://api.deepseek.com/v1`。
2. **密钥 (API Key)**：一串以 `sk-` 开头的长字符串，从 AI 服务商的官网后台申请。这是你的身份凭证，软件会把它存在本地 IndexedDB 里，绝不上传。
3. **模型名称 (Model Name)**：你要调用的具体模型，比如 `deepseek-chat`、`deepseek-reasoner` 等，必须和服务商官网列出的名称完全一致。

填好这三样后，记得点一下那个绿色的小 **Ping 测试** 按钮。如果返回绿色✅和延迟数字，说明电力接通了；如果返回红色❌，鼠标悬停可以看到错误码（比如 401 表示密钥错误，404 表示地址写错，500 表示服务商故障），照着提示修改即可。

**花生**：（恍然大悟地点头）哦！那"灵魂"肯定就是角色卡了对不对？我想让这里有一只知性的核桃姐姐，还有一只无敌可爱的花生妹妹！

**核桃**：完全正确。第二步就是创建角色。

在左侧的侧边栏找到 **【角色】** 图标（那个带着小人面具的标签），点击进入角色管理页面，然后点 **「新建角色」** 按钮。在弹出的编辑面板里，依次填写：

1. **名字**：比如"核桃"或"花生"。
2. **头像**：可以选一个 Emoji（比如 🐭、🌰、🧀），也可以上传一张动漫头像图片（软件会自动压缩到合适大小）。
3. **System Prompt（系统提示词）**：这是最核心的字段——在这里写下角色的性格、口吻、外貌、行为约束。写得好不好，直接决定 AI 演得像不像。建议参考本说明书第二章的 XML 结构化格式。

小提示：如果你在别的地方下载了别人做好的 **SillyTavern V2 标准角色卡**（通常是一张内嵌了 JSON 数据的 PNG 图片），直接在角色管理页点 **「导入」** 按钮，选中那张图片，软件会瞬间读取里面的名字、头像和 System Prompt，省去手填的麻烦。

**花生**：（拍着小手）好耶！现在电力有了，核桃姐姐也有了，那我们怎么开始真正的第一场戏呢？

**核桃**：第三步，新建对话。

在侧边栏切换到 **【对话】** 列表，点击那个大大的 **「新建对话」** 按钮。接着，在右侧的聊天主区域上方，你会看到两个角色绑定槽位：

- **角色 A**：绑定为主对话对象（比如"核桃"）。
- **角色 B**：绑定为第二对话对象（比如"花生"），如果你只想和单个角色聊天，把 B 留空即可。

绑定好之后，对话就正式建立了。软件会为这个对话单独开辟一条消息流，所有聊天记录、世界书触发、蒸馏记忆都归属于这条对话。

**花生**：（跃跃欲试，小爪子放在输入框上）最后一步了！我该怎么和核桃姐姐聊天呢？

**核桃**：第四步，发送消息。

看到底部的那个大输入框了吗？在里面打字，比如输入："核桃姐姐，今天中午吃什么？"

重点来了！输入框下方有两个常驻大按钮：**【发送 A】** 和 **【发送 B】**。

- **【发送 A】**：把你的话发给角色 A（核桃）。消息会被包裹成「交互用户的场景输入」送进核桃的上下文，核桃会用翠绿色气泡回复你。
- **【发送 B】**：把你的话发给角色 B（花生）。消息会送给花生，她用紫罗兰色气泡回复。
- **快捷键**：`Shift + Enter` 等同于点击【发送 A】，方便你打字时不用切回鼠标。

如果你两个角色都绑定了，还可以用 **【旁听】** 按钮，让两个 AI 互相说话、你在旁边看戏——不过这个进阶玩法我们留到中级对话再讲。

**花生**：（转头看向屏幕前的玩家）哇，原来这么简单！屏幕前的小伙伴，你学会了吗？快去点开右上角的设置，把电力和灵魂注入进来，和我们说第一句话吧！

**核桃**：最后给你一张速查表，照着做就不会错：

| 步骤 | 操作位置 | 关键动作 |
|------|----------|----------|
| 1. 配 API | ⚙️设置 → 模型配置 | 填 Base URL + API Key + Model Name → Ping 测试 |
| 2. 建角色 | 侧边栏【角色】→ 新建 | 填名字 + 头像 + System Prompt（或导入 SillyTavern 卡） |
| 3. 建对话 | 侧边栏【对话】→ 新建 | 绑定角色 A（必填）+ 角色 B（可选） |
| 4. 发消息 | 底部输入框 | 打字 → 点【发送 A】或按 Shift+Enter |

## 五、预设对话 1：基础交互与界面认知（初级功能学习）

> 🟢 难度：初级 | 目标：掌握气泡操作（编辑、重试、分支、删除）、界面主题、移动端适配

## 六、预设对话 2：世界书、蒸馏与深度认识（中级功能学习）

> 🟡 难度：中级 | 目标：掌握世界书按需注入、记忆蒸馏、双角色互相认识

**核桃**：（推了推单片眼镜，翻开记录板）记录，测试周期第二天。今天我们进入"中级功能"环节。花生，你有没有发现，昨天我们聊了那么多，但今天再问起昨天的细节，AI 似乎已经记不太清了？

**花生**：（歪头）对呀！我明明告诉过核桃姐姐我最喜欢吃极品奶酪，可是聊着聊着，核桃姐姐好像就忘了！是不是 AI 的记忆力只有七秒，像金鱼一样？

**核桃**：不完全是。AI 模型本身有上下文窗口限制（比如 8K、32K、128K tokens），超出窗口的早期消息会被"挤出"上下文，AI 自然就"忘"了。这就是为什么我们需要 **世界书** 和 **蒸馏** 这两套机制——前者负责"按需想起"，后者负责"压缩记忆"。

**花生**：世界书？是像图书馆那样吗？

**核桃**：可以这么理解，但更聪明。世界书是一堆带关键词的"小抄条目"，平时不占上下文，只有当聊天内容里出现了关键词或别名时，对应条目才会被"激活"并注入到当前上下文里。

举个例子：我们预设的世界书里有一条叫"鼠族耳朵"，关键词是"鼠耳"，别名是"Hamsterkin Ears, 鼠鼠的耳朵, 绒毛耳朵"。平时这条目静静躺在 IndexedDB 里，不消耗任何 token。但只要你聊天时打出"鼠耳"或"鼠鼠的耳朵"这几个字，系统就会在下一轮 AI 回复前，把这条目内容悄悄塞进上下文，让 AI"想起"鼠耳的设定。

**花生**：哇！那如果我一直聊"鼠耳"，它会不会每次都塞一遍，变成复读机？

**核桃**：好问题。系统内置了 **冷却机制 (Cooldown)**：每个条目被激活后，会进入冷却期，冷却期内即使再次命中关键词也不会重复注入。冷却时长按公式 `max(1, floor(最近轮数 / 3))` 动态计算——也就是说，对话越长，冷却越宽松；对话越短，冷却越严格。这样既保证关键设定不丢，又避免复读。

另外，世界书还有 **优先级 (Priority)** 机制：每个条目可以设 1-10 的优先级，数值越大越优先注入。当总 token 预算紧张时，系统会先注入高优先级条目，低优先级的会被跳过。所以核心设定（比如角色身份）给 9-10，次要设定（比如某个地名）给 5-8 即可。

**花生**：那"蒸馏"又是什么？听起来像炼金术！

**核桃**：差不多。**蒸馏 (Distillation)** 是把长对话压缩成"记忆结晶"的机制。

想象一下：你和我聊了 200 轮，前 150 轮的细节 AI 已经记不住了。这时系统会自动触发蒸馏——把那些即将被挤出上下文的旧消息，调用 AI 总结成一段精炼的"记忆摘要"，以琥珀色气泡的形式插入到对话流里。之后，这段摘要会替代原始的几十条消息，持续参与上下文，让 AI"记得"发生过什么，但只占很小的 token。

| 蒸馏触发条件 | 说明 |
|--------------|------|
| 自动触发 | 当对话轮数超过设定阈值（默认 20 轮）且 token 接近上限时，系统自动蒸馏最早的若干轮 |
| 手动触发 | 输入框上方工具栏点【蒸馏】按钮，立即对当前对话执行一次蒸馏 |

蒸馏后的记忆结晶会带一个特殊标记，AI 在回复时会"看到"这段结晶，从而保持长期一致性。如果你觉得某段记忆很重要、想强制让 AI 注意，可以用工具栏的 **【🧬 植入记忆】** 按钮，它会一次性把最新蒸馏记忆 + 状态书注入到下一条消息的上下文里，相当于给 AI"提个醒"。

**花生**：（眼睛亮起来）太棒了！那如果我想让核桃姐姐和花生妹妹"认识彼此"呢？比如让核桃姐姐知道我是喜欢吃奶酪的贵族鼠族，让我知道核桃姐姐是科研鼠族？

**核桃**：这正是 **【🤝 互相认识】** 功能的用途。当两个角色都绑定后，点这个按钮，系统会：

1. 并发地把"角色 A 的角色卡"发给角色 B 的 AI，让它观察并写一段对 A 的"第一印象"。
2. 同时把"角色 B 的角色卡"发给角色 A 的 AI，让它写对 B 的印象。
3. 把这两段印象分别存入世界书，作为"角色 A 对 B 的认知"和"角色 B 对 A 的认知"条目。
4. 之后每次聊天，这些印象条目会按需注入，让两个 AI"记得"对方是谁、有什么特点。

这套机制让双角色对话不再只是"两个 AI 各说各话"，而是真正"认识彼此"的关系网。比如你之后说"花生又偷吃奶酪了"，核桃的 AI 会自动想起"花生是喜欢吃奶酪的贵族鼠族"，回复就会更贴合人设。

**花生**：（抱紧奶酪）才不是偷吃呢，是光明正大地吃！不过这个功能好厉害，那我去试试看！

**核桃**：去吧。记住，世界书负责"按需想起"，蒸馏负责"压缩长记"，互相认识负责"建立关系"——这三件套是中级玩家的核心武器。

## 七、预设对话 3：状态书与 Galgame 数值引擎（高级功能学习 · 数值篇）

> 🔴 难度：高级·数值篇 | 目标：掌握状态书吸附卡、Galgame 数值引擎、非对称信息注入

**核桃**：（神情变得严肃，单片眼镜反光）记录，测试周期第三天。今天进入"高级功能"环节，主题是数值与状态。花生，你玩过恋爱养成游戏吗？就是那种有"好感度""心情值"数值条的游戏。

**花生**：（兴奋地点头）玩过玩过！我最喜欢看着好感度条一点点涨上去，然后触发特殊剧情！可是……AI 聊天里怎么搞数值呀？AI 又不会自己算数。

**核桃**：这就是 **状态书 (State Book)** 和 **Galgame 数值引擎** 要解决的问题。状态书是一张"吸附在对话顶部的状态卡"，它记录当前对话的关键状态信息——比如角色的心情、位置、持有物品、好感度等。每次 AI 回复后，系统会自动更新这张卡，让状态在长对话里保持一致。

在软件里，状态书有两种模式：

| 模式 | 触发方式 | 适用场景 |
|------|----------|----------|
| **手动状态书** | 玩家自己写状态内容，或用工具栏【📜状态书】按钮让 AI 生成 | 简单状态追踪，比如"当前场景：酒馆大厅" |
| **Galgame 数值引擎** | 在角色卡里开启 GalgameCard，AI 每次回复自动输出 JSON 数值 | 恋爱养成、数值追踪、好感度系统 |

**花生**：Galgame 数值引擎？听起来好高级！它是怎么工作的？

**核桃**：它的核心是让 AI 在每次回复的末尾，额外输出一段 JSON 格式的数值数据。系统会用一套 **4 级递进解析器** 把这段 JSON 提取出来，渲染成像素条状的数值面板，吸附在对话顶部。

具体流程是这样的：

1. **上下文构建**：系统在角色卡后面附加一段 Galgame 引擎的默认 prompt，告诉 AI"请在回复末尾输出 `[GALGAME]{"好感度":65,"心情":"开心",...}[/GALGAME]` 格式的数值"。
2. **AI 回复**：AI 正常回复剧情，然后在末尾附上 JSON 数值块。
3. **4 级解析**：系统依次尝试——①直接 JSON.parse；②正则提取 `[GALGAME]...[/GALGAME]` 标签内容再 parse；③容错解析（修复多余换行、缺失引号）；④字段强校验 + 数值 clamp（把越界值拉回合法范围）。
4. **渲染**：解析出的数值渲染成像素条，吸附在对话顶部；JSON 原文从 AI 回复里剥离，不显示给玩家。
5. **非对称注入**：下一轮 AI 回复时，系统把当前数值以"仅 AI 可见"的方式注入上下文——也就是说，玩家看不到这段数值提示，但 AI 能看到，从而根据数值调整剧情走向。

**花生**：非对称注入？就是 AI 看得到、我看不到？

**核桃**：对。这是为了防止"AI 谄媚"——如果数值直接显示在对话里，AI 可能会为了讨好玩家而虚报数值（比如明明好感度该降，却硬说涨了）。非对称注入让数值作为"隐藏提示"传给 AI，AI 必须根据剧情真实地调整数值，玩家只看到渲染后的像素条结果，这样数值才可信。

**花生**：那我能自定义追踪哪些数值吗？

**核桃**：当然可以。在角色卡的 GalgameCard 设置里，你可以修改任意数值字段。

这样，几百条设定几秒钟就能灌进软件，省去手动录入的痛苦。

**花生**：（捡起奶酪，眼睛发亮）太棒了！那导入完世界书，AI 就能自动按设定演了吗？

**核桃**：基本可以，但如果你想当"导演"精细控制剧情走向，还需要学会 **括号引导** 技巧。这是角色扮演玩家圈子里流传的一套"潜规则"——用不同括号包裹的内容，AI 会理解为不同的指令层级：

| 括号类型 | 含义 | 举例 |
|----------|------|------|
| `（）` 中文圆括号 | 旁白 / 动作描写 / 环境描写 | `（窗外下起了大雨，花生缩进被窝）` |
| `「」` 直角引号 | 角色台词 / 强调内容 | `「我才没有害怕呢！」花生嘴硬道。` |
| `【】` 方括号 | 场景标注 / 章节标题 | `【场景：科研塔·深夜】` |
| `（）` + "OOC:" | 出戏指令 / 元指令 | `（OOC: 让核桃突然接到一个紧急电话）` |

其中最实用的是 **OOC（Out Of Character）指令**——当你想跳出剧情、直接给 AI 下"导演指令"时，在消息里用圆括号加 OOC 前缀，AI 会理解这是给它的元指令，而不是剧情的一部分。比如：

> （OOC: 接下来请安排一场意外，让花生把奶酪掉进河里，然后核桃假装不关心其实偷偷帮忙）

AI 收到后会按这个方向推进剧情，但在回复里不会暴露"我收到了指令"这件事，保持沉浸感。

**花生**：（捂嘴）哇，那我岂不是可以偷偷指挥核桃姐姐做任何事？

**核桃**：（推眼镜）……理论上是的。但请注意，OOC 指令要写得具体、明确，避免歧义。模糊的指令比如"（OOC: 让剧情更有趣一点）"AI 很难执行；具体的指令比如"（OOC: 让一个陌生人敲门，自称是花生的远房表亲）"AI 才能精准演绎。

另外，世界书的 **优先级管理** 也很重要。当你导入了几百条设定后，建议按这个原则分级：

| 优先级 | 用途 | 举例 |
|--------|------|------|
| 10 | 角色核心身份、不可遗忘的设定 | "花生是鼠族贵族" |
| 9 | 主要地点、关键道具 | "科研塔""极品奶酪" |
| 7-8 | 次要地点、配角 | "酒馆老板""花园" |
| 5-6 | 背景常识、历史事件 | "百年前的鼠族大迁徙" |
| 1-4 | 极少触发的彩蛋设定 | "某个传说里的神秘宝箱" |

高优先级条目在 token 紧张时优先注入，低优先级会被跳过。所以把"绝对不能忘的"放 9-10，"聊到再想起的"放 5-8，"几乎用不到的"放 1-4。

**花生**：明白了！那我现在就去网上找一份沙盒世界设定 JSON，导入进来当我们的世界观！

**核桃**：很好。记住，Easy 世界书负责"快速建世界"，括号引导负责"精细导剧情"，优先级管理负责"token 预算分配"——这三件套是中高阶玩家的世界塑造武器。

## 九、预设对话 5：角色加固——灵魂小抄、别名魔术与参数的"冰与火之歌"（高级功能学习 · 角色篇）

> 🔵 难度：高级·角色篇 | 目标：掌握 XML 结构化角色卡、别名魔术、采样参数调校

**核桃**：（神情专注，单片眼镜后的目光锐利）记录，测试周期第五天，也是最后一天。今天我们讲"角色加固"——如何让 AI 把一个角色演得入木三分，不崩人设、不出戏。

**花生**：（端正坐好，连奶酪都放下了）我要认真听！怎么才能让核桃姐姐永远知性、让花生永远可爱呢？

**核桃**：核心是三件事：**结构化角色卡**、**别名魔术**、**采样参数调校**。我们一个个讲。

### 9.1 结构化角色卡：XML 是最佳实践

第一件事，角色卡的 System Prompt 不要写成一大段散文，而推荐用 **XML 标签结构化**。XML 标签能让 AI 清晰区分"身份""约束""场景""格式"四个区块，遵守率远高于纯文本。

回顾一下我们预设角色卡的 XML 结构：

```xml
<role>
  [身份、出身、外貌、与其他角色的关系]
</role>

<constraints>
  1. [必须遵守的行为规矩]
  2. [口吻要求]
  3. [特殊反应设定]
</constraints>

<context>
  [当前对话所处的场景或任务]
</context>

<task>
  [输出格式与字数要求]
</task>
```

**为什么用 XML 而不是 Markdown？** 因为大模型在训练时见过大量 XML 格式的结构化数据（网页、文档、配置文件），对 XML 标签的"边界感知"更强，能更准确地把约束条款"记在心里"。

**编写要点**：
- `<role>` 写"是谁"：身份、出身、外貌、关系网。
- `<constraints>` 写"怎么做"：用编号列表，每条一个明确规矩。
- `<context>` 写"在哪"：当前场景、任务背景。
- `<task>` 写"输出什么"：字数、格式、语言风格。
- 总字数控制在 300-500 字，太长挤占上下文预算。

### 9.2 别名魔术：让世界书永不漏触发

第二件事，世界书的 **别名 (Aliases)** 是一套"魔术"——它决定了你的设定会不会被 AI"想起来"。

**花生**：别名不就是同义词吗？有什么魔术的？

**核桃**：魔术在于：玩家在聊天时打出的词，往往是不可预测的。比如你想让 AI 记住"鼠族耳朵"这个设定，但玩家可能打：

- "鼠耳"（简称）
- "鼠鼠的耳朵"（卖萌说法）
- "Hamsterkin Ears"（英文原名）
- "绒毛耳朵"（描述性说法）
- "那对耳朵"（指代说法）

如果你只在关键词里写"鼠耳"，后面四种说法全部不会触发，AI 就"想不起来"设定了。所以别名的魔术是：**把所有可能的同义说法、英文译名、俗称、描述性说法，全部写进 Aliases 字段，用英文逗号分隔**。

| 设定 | 关键词 | 别名（示例） |
|------|--------|--------------|
| 鼠族耳朵 | 鼠耳 | Hamsterkin Ears, 鼠鼠的耳朵, 绒毛耳朵, 那对耳朵 |
| 科研塔 | 科研塔 | 实验室, 研究所, 中央主控室, 那座塔 |
| 极品奶酪 | 极品奶酪 | 大块奶酪, 鼠族美味, 那块奶酪, 牦牛奶酪 |

别名越多，触发率越高，AI 的"记忆"就越完整。但也别滥用——把无关的词写进别名会导致误触发，比如"奶酪"这个别名太宽泛，凡是聊到奶酪都会触发"极品奶酪"条目，反而干扰。所以别名要"精准的同义扩展"，不是"宽泛的语义联想"。

### 9.3 采样参数调校：Temperature 与 Top-P 的冰与火之歌

第三件事，也是最硬核的——**采样参数调校**。在 ⚙️设置 → 模型配置 → 高级参数里，有两个最关键的旋钮：**Temperature（温度）** 和 **Top-P（核采样）**。

**花生**：温度？核采样？听不懂啦！

**核桃**：用奶酪打比方。Temperature 控制 AI 的"脑洞大小"，Top-P 控制 AI 的"用词范围"。

| 参数 | 范围 | 低值效果 | 高值效果 | 角色扮演推荐 |
|------|------|----------|----------|--------------|
| **Temperature** | 0.0 - 2.0 | 保守、确定、重复性强（像背书） | 天马行空、创意强、但容易跑偏 | 0.7 - 1.0 |
| **Top-P** | 0.0 - 1.0 | 只从最可能的词里选（像考试填标准答案） | 从更多候选词里选（像写作文自由发挥） | 0.85 - 0.95 |

**调校口诀**：
- **角色扮演**：Temperature 0.8 + Top-P 0.9 —— 既有创意又不跑偏，是大多数 RP 场景的甜点区。
- **严肃剧情**：Temperature 0.6 + Top-P 0.85 —— 更稳定，适合推理、悬疑、严肃叙事。
- **搞笑日常**：Temperature 1.0 + Top-P 0.95 —— 更放飞，适合轻松搞笑、脑洞剧情。
- **数值引擎**：Temperature 0.3 + Top-P 0.8 —— 数值输出需要确定性，温度太高会导致 JSON 格式错乱。

**冰与火之歌**：Temperature 是"火"，烧得太旺 AI 会胡说八道，烧得太弱 AI 会死板重复；Top-P 是"冰"，冻得太紧 AI 会词穷，化得太开 AI 会啰嗦。两者要配合调节——通常固定一个、调另一个，找到最适合你角色的平衡点。

另外，软件还支持 **reasoning_effort（深度思考）** 参数，在工具栏点【🧠深度思考】开启。开启后 AI 会先在内部"思考"一轮再回复，适合复杂剧情决策、推理场景，但会增加响应延迟和 token 消耗，日常聊天不建议常开。

**花生**：（认真记笔记）Temperature 0.8、Top-P 0.9……记住了！那我现在就去调参数，让核桃姐姐变得更知性！

**核桃**：（满意地点头）很好。记住，结构化角色卡是"灵魂小抄"，别名魔术是"记忆保险"，采样参数是"性格调节旋钮"——这三件套是高级玩家的角色加固武器。

**花生**：（站起来，向屏幕前的玩家鞠躬）屏幕前的小伙伴，五天的教学到这里就结束啦！从配 API 到调参数，你现在已经是 Easy酒馆Pro 的高手了！快去创造你自己的故事吧，吱！

**核桃**：（合上记录板，微笑）记录，教学周期结束。祝你在 Easy酒馆Pro 里玩得开心。如果你忘了任何操作，随时回来翻阅这套预设对话——我们一直在这里。
