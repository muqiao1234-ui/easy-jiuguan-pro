## 1. 快速开始

### 1.1 启动应用

**方式一：开发模式**（推荐用于调试）
```bash
cd "I:/new jg"
npm run dev
```
浏览器访问 `http://localhost:5173`

**方式二：生产模式**（推荐日常使用）
直接双击 `dist/index.html`，或部署到任意静态托管服务。

### 1.2 首次使用流程

```
添加模型通道 → Ping 测试 → 创建角色 A/B → 创建对话 → 开始聊天
```

> ⚠️ **重要**：应用完全运行在浏览器端，所有数据（API Key、聊天记录）存储在浏览器 IndexedDB 中，不会上传到任何服务器。API Key 仅用于你配置的模型服务商。

## 2. 模块 A：多模型通道管理

### 2.1 功能概述

模型通道管理器允许你添加多个 AI API 渠道（DeepSeek、OpenRouter、OpenAI、本地 Ollama 等），并一键测试每个渠道的响应延迟。

### 2.2 界面位置

- **PC 端**：侧边栏 → 点击 **设置选项卡**（齿轮图标）→ 找到「模型通道」区域
- **移动端**：底部 Tab → 点击 **设置图标** → 找到「模型通道」区域

### 2.3 添加模型通道

1. 点击 **「+ 添加」** 按钮
2. 在弹出的表单中填写：

| 字段 | 说明 | 示例 |
|------|------|------|
| **名称** | 自定义显示名称 | `DeepSeek V3`、`本地 Ollama` |
| **Base URL** | API 服务地址 | `https://api.deepseek.com` |
| **API Key** | 你的 API 密钥 | `sk-xxxxxxxx` |
| **默认模型名** | 要调用的模型 ID | `deepseek-chat`、`gpt-4o-mini` |

3. 点击 **「添加」** 保存

### 2.4 Ping 延迟测试

每个模型卡下方有两个操作按钮：

- **「Ping」按钮**：发送一个极简请求（`max_tokens=1, prompt="p", stream=false`）到该渠道，计算往返时间

延迟结果解读：

| 显示 | 含义 |
|------|------|
| `X ms`（绿色，<500ms） | 响应快速，适合实时对话 |
| `X ms`（黄色，500-1500ms） | 中等延迟，可接受 |
| `X ms`（红色，>1500ms） | 延迟较高，可能影响体验 |
| `超时` | 10 秒内无响应，服务器不可达 |
| `Error/CORS` | 跨域错误或网络失败 |

### 2.5 设为当前模型

每个模型卡底部有 **「聊天」** 和 **「蒸馏」** 两个角色按钮：

- **聊天**：将此模型设为当前对话使用的主模型
- **蒸馏**：将此模型设为记忆蒸馏时使用的模型（通常选更便宜/轻量的模型）

分别对应主控台顶部的两个下拉菜单。

### 2.6 编辑和删除

- 点击模型卡右侧的 **✏️ 编辑** 图标修改配置
- 点击 **🗑️ 删除** 图标移除模型

## 3. 模块 B：双角色对话沙盒

### 3.1 功能概述

这是系统的核心。你可以在一个对话中同时与两个 AI 角色互动：

- **发送给角色 A**：你说一句话，角色 A 回复
- **发送给角色 B**：你说一句话，角色 B 回复
- **让角色 B 旁听插话**：不输入新内容，让角色 B 基于当前对话氛围给出观察反应

### 3.2 创建角色

1. 侧边栏 → **角色管理** 选项卡
2. 点击 **「+ 添加」**
3. 填写：

| 字段 | 说明 |
|------|------|
| **角色名** | 角色的名字（如"旅店老板"） |
| **头像** | emoji 表情（如 `🧙`），可输入 1-2 个字符 |
| **System Prompt** | 角色的系统提示词，定义性格、说话风格、知识背景等 |
| **绑定世界书** | （可选）关联一个世界书 |

### 3.3 创建对话

1. 侧边栏 → **对话列表** 选项卡
2. 点击 **「+ 新建」**
3. 填写对话标题，选择角色 A 和角色 B
4. 点击 **「创建」**

### 3.4 聊天界面布局（PC 端，从上到下）

```
┌──────────────────────────────────────────────┐
│ 🍺 Easy酒馆Pro                  [⚙️设置]     │ ← 顶栏
├──────────────────────────────────────────────┤
│ 角色A: [▼ 下拉选择]  角色B: [▼ 下拉选择]     │ ← 角色选择器
├──────────────────────────────────────────────┤
│ 聊天模型: [▼]        蒸馏模型: [▼]           │ ← 模型选择器
├──────────────────────────────────────────────┤
│ 📜 独立状态书（可折叠）                       │ ← 状态书
├──────────────────────────────────────────────┤
│                                              │
│  💬 角色A: "你好，旅行者..."                  │ ← 消息区
│  👤 你: "来一杯麦酒"                          │
│  💬 角色B: "今天的酒不错..."                  │
│  💎 记忆结晶: 第1-20轮摘要...                 │ ← 蒸馏气泡
│                                              │
├──────────────────────────────────────────────┤
│ [输入框................................]     │ ← 输入区
│ [→发送A] [→发送B] [👂B旁听] [🧪蒸馏]        │
└──────────────────────────────────────────────┘
```

### 3.5 三种发送操作

#### 操作 1：发送给角色 A

1. 在输入框输入文本
2. 点击 **「→ 发送给角色A」**（或按 `Enter`，默认快捷键）
3. 系统流程：
   - 保存你的消息到数据库
   - 扫描世界书（匹配关键词）
   - 组装 Context：System Prompt → 状态书 → 世界书条目 → 最近 M 轮对话 → 蒸馏摘要
   - 调用聊天模型的 SSE 流式 API
   - **逐字实时渲染**角色 A 的回复
   - 保存回复到数据库，更新对话树

#### 操作 2：发送给角色 B

与操作 1 完全相同，但目标角色是角色 B。使用角色 B 的 System Prompt 和通用 Context。

#### 操作 3：让角色 B 旁听插话

1. 点击 **「👂 让角色B旁听」**（不需要输入文本）
2. 角色 B 收到最近对话上下文 + 特殊指令
3. 角色 B 生成**一条**观察/反应后停止
4. 以紫色气泡显示在对话中

> **设计意图**：这是一个"戏剧性打断"机制——角色 B 不会自动插话（防止抢话），必须你手动触发。每次点击只产生一条反应。

### 3.6 消息气泡样式

| 角色 | 颜色 | 对齐 | 前缀标签 |
|------|------|------|----------|
| **你（用户）** | 蓝色 | 右对齐 | `👤 你` |
| **角色 A** | 绿色 | 左对齐 | `💬 角色A名` |
| **角色 B** | 紫色 | 左对齐 | `💬 角色B名` |
| **系统消息** | 灰色 | 居中 | `⚙️ 系统` |
| **记忆结晶** | 琥珀色虚线边框 | 居中 | `💎 记忆结晶` |

### 3.7 对话树与分支

每条消息（除蒸馏消息外）都支持分支：

1. **创建分支**：鼠标悬停在角色消息气泡上，点击右上角出现的 **✂️ 分支** 按钮
2. **分支效果**：当前活跃指针切换到该节点，后续新消息从该节点分叉
3. **分支树**：在消息时间戳下方可以看到分支历史

> 对话采用**树状链表**结构：每个 MessageNode 通过 `parentId` 指向上级，`childrenIds` 记录分叉。`currentActiveNodeId` 指向当前活跃叶子。

### 3.8 中断流式传输

在 AI 回复过程中，输入框下方会出现 **「⏹ 停止」** 按钮，点击即可中断当前请求。中断后已接收的部分内容会保留。

## 4. 模块 C：记忆蒸馏引擎

### 4.1 功能概述

记忆蒸馏引擎是系统的"节流与对抗失忆"核心。当对话积累到一定轮数后，将历史对话压缩为摘要，释放上下文窗口空间。

### 4.2 工作流程

```
对话积累 N 轮 → 触发蒸馏 → 调用蒸馏模型 → 生成摘要 →
创建 💎 记忆结晶气泡 → 归档原始消息（isArchived=true）
```

### 4.3 手动触发蒸馏

1. 点击输入框下方的 **「🧪 蒸馏」** 按钮
2. 系统获取当前对话所有未归档消息
3. 调用**蒸馏模型**（你在模型选择器中指定的那个）
4. 蒸馏 Prompt：
   ```
   请将以下对话内容浓缩为简洁的摘要，
   保留关键情节、人物态度变化和重要信息：
   {N 轮对话文本}
   ```
5. 生成蒸馏结果，以 **💎 记忆结晶** 气泡插入对话
6. 原始消息被标记为 `isArchived = true`，后续 Context 组装时**不再包含**

### 4.4 自动触发蒸馏

在设置中开启「自动触发蒸馏」后：
- 每次发送消息后，系统检查未归档消息数是否 ≥ 触发阈值（默认 10 轮）
- 若满足条件，后台自动执行蒸馏（不阻塞对话）

### 4.5 蒸馏结果操作

每条记忆结晶气泡提供两个操作按钮：

- **📋 复制**：将蒸馏文本复制到剪贴板
- **➕ 插入上下文**：将此条蒸馏摘要强制加入后续对话的 Context 中

### 4.6 蒸馏→世界书

如果需要将蒸馏内容写入世界书：
1. 先点击记忆结晶气泡的「复制」按钮
2. 手动切换到世界书管理界面
3. 创建新条目，设置触发关键词
4. 粘贴蒸馏文本到「注入内容」字段

### 4.7 蒸馏浓度参数

| 浓度 | 蒸馏效果 |
|------|----------|
| 1-3 | **极简事实**：仅保留关键事件和人物 |
| 4-7 | **标准摘要**：事件 + 情感变化 |
| 8-10 | **详细大纲**：包含细节和对话片段 |

> 注：蒸馏浓度参数在设置面板中调整，当前版本通过滑块调节。

## 5. 模块 D：世界书扫描器

### 5.1 功能概述

世界书（Lorebook）是一个关键词触发的知识库。当对话中出现特定关键词时，对应的世界设定会自动注入到 AI 的上下文中，让角色"记住"世界观。

### 5.2 创建世界书

1. 侧边栏 → **世界书** 选项卡
2. 点击 **「+ 新建」**
3. 输入世界书名称（如 `酒馆世界观`、`魔法设定`）

### 5.3 添加条目

在世界书卡片中点击 **「+」** 按钮，填写：

| 字段 | 说明 | 示例 |
|------|------|------|
| **触发关键词** | 逗号分隔的关键词列表 | `麦酒, 啤酒, ale` |
| **注入内容** | 匹配时注入的世界设定 | `麦酒是本店招牌，由矮人山脉的大麦酿造，售价 5 铜币一杯` |
| **优先级** | 1-10，数字越大越优先注入 | `7` |

### 5.4 扫描机制

**触发时机**：每次发送消息给 AI 之前（Context 组装阶段）

**扫描范围**：最近 M 轮未归档对话（M 可在设置中调整，默认 20）

**匹配算法**：
```
对世界书的每个条目：
  对条目的每个关键词：
    使用 String.includes() 和 RegExp 进行不区分大小写匹配
    匹配到任意一个关键词 → 该条目被选中
按 priority 降序排序
取前 3 条（可配置）
```

**注入格式**：
```
[世界设定: 麦酒] 麦酒是本店招牌，由矮人山脉的大麦酿造，售价 5 铜币一杯
```

### 5.5 绑定世界书到角色

在创建/编辑角色时，可以从下拉菜单中选择关联的世界书。每个角色可绑定不同世界书。

### 5.6 条目管理

- **编辑**：修改关键词、内容或优先级
- **删除**：移除条目或整个世界书

## 6. 模块 E：独立状态书

### 6.1 功能概述

独立状态书是一个"全知状态看板"——一段可手动编辑的文本，描述当前客观环境、好感度动态等信息。它被强制固定在每次 AI 请求的 Context 中，防止设定漂移。

### 6.2 访问方式

**PC 端**：侧边栏 → **状态书** 选项卡

**聊天界面**：消息列表顶部的 **「📜 独立状态书」** 可折叠面板

### 6.3 使用示例

```
当前地点：老橡木酒馆一楼大厅，烛光昏暗，壁炉噼啪作响
时间：初冬傍晚，外面飘着小雪，气温寒冷
角色A对用户的好感度：友好（65/100），因用户上次帮忙赶走了闹事者
角色B的情绪：疑心重重，对陌生人保持警惕
```

### 6.4 Context 注入位置

状态书内容被**固定**在 messages 数组的第二项（紧跟 System Prompt 之后）：

```
messages = [
  { role: 'system', content: '角色A的System Prompt' },
  { role: 'system', content: '[独立状态书] 当前地点：酒馆...' },  ← 这里
  { role: 'system', content: '[世界设定: xxx] ...' },
  { role: 'user', content: '你: 你好' },
  ...
]
```

> **设计理由**：状态书紧跟在 System Prompt 之后，确保 AI 在生成回复时优先参考当前环境状态，而非被长对话历史冲淡。

### 6.5 保存机制

- **失焦自动保存**：编辑后点击其他区域，自动写入 IndexedDB
- **手动保存**：点击「保存」按钮
- 每次对话切换时，自动加载对应对话的状态书

## 7. 设置面板

### 7.1 访问方式

侧边栏 → **设置** 选项卡（齿轮图标）

### 7.2 记忆蒸馏设置

| 参数 | 默认值 | 范围 | 说明 |
|------|--------|------|------|
| **触发阈值** | 10 轮 | 5-50 | 未归档消息达到此数量后触发蒸馏 |
| **自动触发** | 关闭 | 开关 | 开启后达到阈值自动蒸馏 |
| **蒸馏浓度** | 5 | 1-10 | 1=极简事实，10=详细大纲 |

### 7.3 上下文配置

| 参数 | 默认值 | 范围 | 说明 |
|------|--------|------|------|
| **最近轮数 (M)** | 20 | 5-50 | 每次发送时携带的最近对话轮数 |
| **最大蒸馏节点 (N)** | 5 | 1-15 | 每次发送时注入的蒸馏摘要条数 |

### 7.4 数据管理

- **导出数据**：将所有数据（模型、角色、对话、消息、世界书、状态书）导出为 JSON 文件
- **导入数据**：从 JSON 文件恢复数据（会**覆盖**现有数据）

> 建议定期导出备份，浏览器清除数据会导致 IndexedDB 内容丢失。

## 8. 数据管理

### 8.1 存储架构

全部数据存储在浏览器 IndexedDB 中，通过 `localForage` 库提供异步 API。**严禁使用 localStorage**（5MB 限制），IndexedDB 存储容量通常为浏览器可用磁盘空间的 50-60%。

### 8.2 六个数据仓库

| 仓库 | localForage Key | 存储内容 |
|------|-----------------|----------|
| models | `tavern_models` | API 渠道配置（含 API Key） |
| characters | `tavern_characters` | 角色定义（System Prompt + 头像） |
| conversations | `tavern_conversations` | 对话元数据（标题 + 活跃节点指针） |
| message_nodes | `tavern_message_nodes` | 所有消息（树状链表结构） |
| worldbooks | `tavern_worldbooks` | 世界书及其条目 |
| global_states | `tavern_global_states` | 每个对话的独立状态书文本 |

### 8.3 数据安全

- **API Key** 存储在 IndexedDB 中，明文存储
- 所有数据仅在浏览器本地，不会同步到云端
- 清除浏览器数据会**永久丢失**所有对话记录
- **强烈建议定期导出备份**

### 8.4 导入/导出格式

```json
{
  "models": [{ "id": "uuid", "name": "DeepSeek", "baseUrl": "...", "apiKey": "sk-...", "defaultModel": "deepseek-chat", "latency": 340 }],
  "characters": [{ "id": "uuid", "name": "旅店老板", "avatar": "🧙", "systemPrompt": "你是...", "worldBookId": "..." }],
  "conversations": [{ "id": "uuid", "title": "酒馆夜谈", "currentActiveNodeId": "uuid", "characterAId": "...", "characterBId": "..." }],
  "message_nodes": [{ "id": "uuid", "conversationId": "...", "parentId": null, "childrenIds": [], "role": "user", "senderName": "你", "content": "你好", "isArchived": false, "timestamp": 1234567890 }],
  "worldbooks": [{ "id": "uuid", "name": "酒馆世界观", "entries": [{ "id": "uuid", "keys": ["麦酒"], "value": "...", "priority": 7 }] }],
  "global_states": [{ "conversationId": "uuid", "scribeContent": "当前地点：酒馆..." }]
}
```

移动端侧边栏内容通过底部 Tab 切换，非对话视图时内容显示在顶部区域。

## 10. 常见问题

### Q1：为什么添加模型后无法发送消息？

检查以下条件是否全部满足：
- ✅ 已添加至少一个模型，并设为「聊天模型」
- ✅ 已创建至少两个角色（角色 A 和角色 B）
- ✅ 已创建对话并选择了角色 A 和角色 B
- ✅ 当前对话处于选中状态

### Q2：为什么 Ping 显示 Error/CORS？

浏览器安全策略限制。如果 API 服务商未配置 CORS 头，前端直连会失败。解决方案：
- 使用支持 CORS 的服务商（如 OpenRouter）
- 通过本地代理转发（如 Ollama 本地部署）
- 检查 API Key 是否正确

### Q3：蒸馏后对话变少了？

这是正常行为。蒸馏机制会将历史消息标记为 `isArchived=true`，这些消息在 Context 组装时被跳过——但它们**不会从数据库中删除**。如果你需要查看完整历史，可以导出数据。

### Q4：世界书条目没有被触发？

可能原因：
- 关键词拼写不匹配（区分大小写取决于是否使用正则）
- 世界书未绑定到目标角色
- 扫描的 M 轮范围内没有包含该关键词
- 同一轮触发条目超过 3 条，低优先级条目被挤出

### Q5：数据丢失了怎么办？

- 检查是否清除了浏览器缓存/Cookie
- 是否使用了浏览器隐私模式（关闭后清除）
- **预防**：定期使用设置中的「导出数据」功能备份

### Q6：Service Worker 注册失败？

当前 SW 为占位，不影响核心功能。PWA 离线缓存功能后续版本完善。
